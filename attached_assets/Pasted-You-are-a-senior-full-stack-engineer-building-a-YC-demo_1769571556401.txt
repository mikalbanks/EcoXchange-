NEW: Stablecoin Ledger (SIMULATED)
LedgerAccount:
- id, userId, label (e.g., “USDC Demo Wallet” or “USD Ledger”), currency (USDC_DEMO or USD_LEDGER), createdAt
LedgerEntry:
- id, accountId, type (DEPOSIT/WITHDRAWAL/RESERVE/RELEASE/PAYOUT), amount, referenceType (COMMITMENT/DISTRIBUTION), referenceId, memo, createdAt

Ledger Rules:
- In DEMO_STABLECOIN=true mode:
  - Each investor gets a LedgerAccount in currency=USDC_DEMO
  - Seed investor starts with DEPOSIT of 250,000 USDC_DEMO
  - When investor invests and is eligible (KYC APPROVED + accredited):
      Create Commitment status=CONFIRMED AND create LedgerEntry type=RESERVE to earmark funds
  - If investor cancels:
      Create LedgerEntry type=RELEASE and set Commitment=CANCELED
  - When Distribution marked PAID:
      Create LedgerEntry type=PAYOUT into investor accounts based on DistributionPayout amounts
- In DEMO_STABLECOIN=false mode:
  - Do not show reserve/release/payout widgets; still write commitments/distributions records
  - LedgerAccount can exist with USD_LEDGER but optional (keep it simple)

BUSINESS LOGIC:
- Only ISSUER can create projects/offerings for self
- Only OPEN offerings appear in marketplace
- Only INVESTOR with kycStatus=APPROVED and accredited=true can make CONFIRMED commitments
- Minting tokens requires offering CLOSED and not already minted
- Token allocations proportional to total confirmed commitments
- Distribution payouts proportional to token allocations (or commitments if tokens not minted yet)
