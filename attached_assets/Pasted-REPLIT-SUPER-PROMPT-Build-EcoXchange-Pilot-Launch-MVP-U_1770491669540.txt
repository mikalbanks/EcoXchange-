REPLIT SUPER-PROMPT — Build EcoXchange Pilot Launch MVP (US-only, Solar-first)

You are a senior full-stack engineer + product builder. Build a production-quality MVP for “EcoXchange”: a platform that takes messy small renewable project inputs and turns them into an investor-ready deal package with (1) standardized project profile, (2) readiness/compliance score (green/yellow/red + reasons), (3) capital stack builder (tax credits + equity now; debt placeholder), (4) investor matching + soft commitments, and (5) a project data room + basic doc generation. PRIMARY GOAL (Pilot MVP)
- Developers can submit a project via wizard → platform generates:
  - canonical project profile
  - readiness score + flags

- Investors can browse approved projects, filter, view score + docs, and “Commit Interest” (soft commitment/LOI intent).
- Admin can review/approve projects, adjust score overrides, manage users, and export project packets.

NON-GOALS (Do NOT build in MVP)
- No secondary trading
- No tokenization/blockchain
- No real tax credit marketplace transactions
- No deep interconnection modeling
- No full payment rails for actual securities funding (only “deposits” optional; default is soft commitments)

ASSUMED USERS / ROLES

READINESS SCORING ENGINE (rules-based MVP)
Implement computeReadiness(project, documents, checklist, capitalStack) -> ReadinessScore

Start with a 100-point base, subtract for risk/missing:
- Site control:
  - NONE: -25 (RED flag)
  - LOI: -15
  - OPTION: -8
  - LEASE/OWNED: 0
- Interconnection:
  - UNKNOWN: -20
  - APPLIED: -15
  - STUDY: -10
  - IA_EXECUTED: -3
  - READY_TO_BUILD: 0
- Permitting:
  - UNKNOWN: -15
  - IN_PROGRESS: -10
  - SUBMITTED: -5
  - APPROVED: 0
- Offtaker:
  - MERCHANT: -12
  - COMMUNITY_SOLAR: -6
  - C&I: -4
  - UTILITY: -2
- Documents completeness:
  - For each required checklist item MISSING: -3 (cap at -24)
- Tax credit readiness:
  - taxCreditEstimated missing/0: -8
  - transferabilityReady false: -6 (not fatal)
- FEOC risk (MVP = developer attestation checkbox + doc upload placeholder):
  - if developer checked “FEOC unknown”: -8
  - if FEOC attestation uploaded: 0

Rating thresholds:
- GREEN: score >= 75 and no fatal flags
- YELLOW: score 50–74 OR minor fatal risk
- RED: score < 50 OR any fatal flag (siteControl NONE, or interconnection UNKNOWN + stage=NTP/CONSTRUCTION)

Return:
- reasons: human-readable list of the top 5 deductions + what to fix
- flags: booleans for each major risk

CAPITAL STACK ENGINE (MVP)
