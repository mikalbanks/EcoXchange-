3) Support Persona webhooks to update status automatically.
4) Keep it demo-safe: read-only KYC, no payments, no custody.

ASSUMPTIONS
- EcoXchange already has authentication and role-based users: Admin, Issuer, Investor.
- EcoXchange has a backend API and a database (use existing ORM/models if present).

IMPLEMENTATION REQUIREMENTS
A) DATABASE / MODEL CHANGES
- Add fields to the existing User model/table (or create a PersonaProfile table linked to User):
  - persona_inquiry_id (string, nullable)
  - persona_status (string enum: 'not_started' | 'pending' | 'completed' | 'failed' )
  - persona_verified_at (datetime, nullable)
  - persona_last_event_at (datetime, nullable)
  - persona_payload (json, nullable)  // store minimal webhook payload for debugging (no sensitive images)
Default persona_status = 'not_started'

B) BACKEND ENDPOINTS
1) POST /api/persona/inquiry
- Auth required
- Creates a Persona Inquiry for the logged-in user using Persona REST API:
  - template_id = PERSONA_TEMPLATE_ID
  - reference_id = user.id
  - optional fields: name/email if available
- Save persona_inquiry_id + set persona_status='pending'
- Return: { inquiryId, sessionToken OR client configuration needed for embedded flow }
NOTE: Prefer embedded flow approach supported by Persona (use their recommended server call to create inquiry and return a one-time token/config needed by the client). If Persona requires a different object (e.g., "inquiry session"), follow their docs.

2) GET /api/persona/status
- Auth required
- Returns current persona_status and persona_verified_at for the logged-in user.

3) POST /api/persona/webhook
- Public endpoint
- Verify webhook signature using PERSONA_WEBHOOK_SECRET (reject if invalid)
- Parse event type and update the correct user using reference_id or inquiry_id mapping
- Update:
  - persona_status = 'completed' if inquiry approved/complete
  - persona_status = 'failed' if declined/failed
  - persona_last_event_at
  - persona_verified_at when completed
- Respond 200 quickly

- Shows status badge:

- Button behavior:
